What AFC Does

Pre-Execution Interception

Every tool call is evaluated before it executes. Dangerous commands like curl | sh are blocked in real time — not detected after the fact.

Prompt Injection Detection

Scans untrusted input for injection patterns. Even if the agent is tricked, the external policy boundary prevents real harm from occurring.

Approval Workflow

High-risk operations require admin approval. Files like ~/.ssh/id_rsa trigger a pause — the agent waits until you say yes or no.

One-Click Quarantine

Isolate a device instantly. Shell, network, and plugin operations are all blocked. The device continues to report status but can do no harm.

Offline Resilience

Policy is cached locally. Even when the Control Center is unreachable, the Sidecar enforces the last known policy. Events queue up and replay on reconnect.

Full Audit Trail

Every tool call, every policy decision, every admin action is recorded. Answerable. Auditable. Explainable.

Threat Model

Five attack surfaces that AFC protects against

🪝

Prompt Injection

Malicious instructions embedded in web pages, emails, or tool results

💣

Dangerous Shell

Remote script execution, privilege escalation, system destruction

📤

Data Exfiltration

Sensitive files sent to external domains, email, or pastebin

🧬

Persistence

Agent memory poisoning, plugin install, cron job creation

🔄

Policy Drift

Local configuration tampering to escape centralized governance
